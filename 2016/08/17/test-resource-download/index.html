Test Resource Download

Thought

Suddently I wounder, though it’s possiable to load a image resource from local static website on GitHub Pages, if is the same possiable to download other resource via GitHub Pages with Hexo, So I did some test.

Start Test

  • I created a new article using hexo new post_name, and it seem automatically built a new folder which name same as the article (why?).
  • So I directly make a file into this folder, for most comment situation, I choose a archive file, inside it is a text file with a sentence resource test.
  • Then edit the markdown file, add a link [resource](test-resource-download/file.zip) like what image resource did, start hexo local server and check the result (resource).
  • And it fail, click from home link, it shown:
  • Click from the article, it shown:
  • It seem the folder name is redundancy, next try, I change into [resource](file.zip) resource.
  • Restart Hexo server, try again, and find that click from home it shown no doubtly:
  • And from the article site, the links is:
    http://localhost:4000/2016/08/17/test-resource-download/file.zip
    This is a correct link.
  • One more try, deploy on remote server, use hexo g, hexo d .
  • And as expected, it react correct in the post site, download the archive file is avaliable from the artilcle.

Summary

  • Download resource can be done, just use the link as [resource](file.zip), dont’t add with folder name.
  • And fail to acquire true link from the home page, I think it’s hexo’s problem (I’m not mean that’s a BUG), hexo has special processing image link, so I should write a plugin about this or using JavaScripts to correct this(but I’m not familiar with this both, I have not much interested about this now :/).
